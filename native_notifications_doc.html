Native Attendance Reminders

Line-by-Line Dart & Kotlin Deep Dive

⚡ 1. NotificationService.dart

This is the "Commander". It tells the Native OS what to do and checks if any work was done in the background.

class NotificationService {
  // 1. The "Pipe": This is the bridge that connects Dart to Native code
  static const _channel = MethodChannel('com.attendance.record_attendance/notifications');

  // 2. The Orders: This function translates Flutter settings into Native alarms
  Future<void> refreshAlarms() async {
    final settings = _ref.read(settingsProvider);
    
    // 3. SYNC: Save the hours to shared memory (XML on Android)
    final prefs = await SharedPreferences.getInstance();
    await prefs.setInt('flutter.check_in_hour', settings.checkInTime.hour);
    await prefs.setInt('flutter.check_in_minute', settings.checkInTime.minute);

    // 4. COMMAND: Call the Native side and pass the hours as a map
    await _channel.invokeMethod('scheduleAlarms', {
      'checkInHour': settings.checkInTime.hour,
      'checkInMinute': settings.checkInTime.minute,
      'checkOutHour': settings.checkOutTime.hour,
      'checkOutMinute': settings.checkOutTime.minute,
    });
  }

  // 5. THE LOOP: This checks if the user clicked "Check In" on a notification
  Future<void> checkPendingActions() async {
    final prefs = await SharedPreferences.getInstance();
    final bool pendingCheckIn = prefs.getBool('flutter.pending_native_check_in') ?? false;

    // 6. If a click was recorded by Native, finish the work in Flutter
    if (pendingCheckIn) {
      await _ref.read(attendanceRecordsProvider(DateTime.now()).notifier).checkIn();
      await prefs.setBool('flutter.pending_native_check_in', false); // Reset the flag
    }
  }
}

📅 2. Attendance Provider Sync

Every time you check-in manually inside the app, we must tell the Native OS so it stops showing reminders for today.

Future<void> checkIn() async {
  // ... (Normal SQLite logic here) ...

  // 1. Tell Native "User is safe!"
  final prefs = await SharedPreferences.getInstance();
  await prefs.setBool('flutter.is_checked_in', true);
  await prefs.setBool('flutter.is_checked_out', false);
  
  // Now, when the Native AlarmReceiver wakes up at 9:00 AM,
  // it will read "flutter.is_checked_in" == true and DO NOTHING.
}

🤖 3. AlarmReceiver.kt (Android Brain)

This Kotlin class wakes up and reads the "orders" left by the Dart Commander above.

class AlarmReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        // 1. Open shared memory (SharedPreferences)
        val prefs = context.getSharedPreferences("app_prefs", Context.MODE_PRIVATE)
        
        // 2. Read the "Is Safe" flag we saved from Dart above
        val isCheckedIn = prefs.getBoolean("is_checked_in", false)
        
        // 3. If Dart said "True", then don't show any notification
        if (isCheckedIn) return
        
        // ... otherwise show the sticky popup ...
    }
}
